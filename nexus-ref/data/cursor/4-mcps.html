What Is MCP & How It Works
Overview

Model Context Protocol (MCP) connects Cursor to external tools and data sources. Instead of explaining your project structure repeatedly, integrate directly with GitHub, Slack, databases, and 100+ other services.

Write MCP servers in any language that can print to stdout or serve an HTTP endpoint — Python, JavaScript, Go, Rust, etc.

Key IdeaMCP servers expose capabilities (tools, prompts, resources) through a standard protocol. Cursor discovers and calls them automatically when relevant.
Protocol Capabilities
FeatureStatusDescription
ToolsSupportedFunctions the AI model can execute
PromptsSupportedTemplated messages and workflows for users
ResourcesSupportedStructured data sources that can be read/referenced
RootsSupportedServer-initiated inquiries into URI/filesystem boundaries
ElicitationSupportedServer-initiated requests for additional info from users
Full SpecCursor supports all five core MCP capabilities. Most servers only expose Tools, but Resources and Prompts are powerful for richer integrations.
Transport Methods
Three Ways to Connect

stdio

  • Runs locally
  • Cursor manages the process
  • Single-user only
  • Input: shell command
  • Auth: manual (env vars)
  • Best for local dev tools

SSE (Server-Sent Events)

  • Local or remote
  • Deploy as a server
  • Multi-user capable
  • Input: URL to SSE endpoint
  • Auth: OAuth supported
  • Best for shared/cloud tools

Streamable HTTP

  • Local or remote
  • Deploy as a server
  • Multi-user capable
  • Input: URL to HTTP endpoint
  • Auth: OAuth supported
  • Best for modern HTTP APIs
Transport ChoiceUse stdio for local-only, private tools. Use SSE or Streamable HTTP when deploying for teams or connecting to remote services. Note: envFile is only available for stdio servers.
Configuration & mcp.json
Config Locations
Project Config (shared with team) .cursor/ mcp.json # project-specific tools Global Config (personal, all projects) ~/.cursor/ mcp.json # available everywhere
PriorityProject-level .cursor/mcp.json is ideal for team-shared tools. Global ~/.cursor/mcp.json is for personal utilities available across all projects.
Config Interpolation Variables
VariableResolves To
${env:NAME}Environment variable value
${userHome}Path to your home folder
${workspaceFolder}Project root (contains .cursor/mcp.json)
${workspaceFolderBasename}Name of the project root folder
${pathSeparator}OS path separator (/ or \)
${/}Alias for path separator
Use Variables for SecretsAlways use ${env:API_KEY} instead of hardcoding secrets. Works in command, args, env, url, and headers fields.
stdio Server Config
mcp.json — Node.js CLI server 
{
  "mcpServers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "mcp-server"],
      "env": {
        "API_KEY": "${env:API_KEY}"
      }
    }
  }
}
mcp.json — Python server 
{
  "mcpServers": {
    "server-name": {
      "command": "python",
      "args": ["mcp-server.py"],
      "env": {
        "API_KEY": "${env:api-key}"
      }
    }
  }
}
FieldRequiredDescription
commandYesExecutable on PATH or full path
argsNoArray of arguments
envNoEnvironment variables for the server
envFileNoPath to .env file (stdio only!)
Remote Server Config
mcp.json — HTTP/SSE remote server 
{
  "mcpServers": {
    "server-name": {
      "url": "http://localhost:3000/mcp",
      "headers": {
        "API_KEY": "${env:MY_TOKEN}"
      }
    }
  }
}
mcp.json — with static OAuth 
{
  "mcpServers": {
    "oauth-server": {
      "url": "https://api.example.com/mcp",
      "auth": {
        "CLIENT_ID": "${env:MCP_CLIENT_ID}",
        "CLIENT_SECRET": "${env:MCP_CLIENT_SECRET}",
        "scopes": ["read", "write"]
      }
    }
  }
}
Remote ServersRemote servers (HTTP/SSE) do not support envFile. Use config interpolation with environment variables set in your shell profile instead.
Authentication & OAuth
Auth Methods Overview
Environment Variables
Pass API keys via env field or envFile. Use ${env:NAME} interpolation. Works for all transport types.
OAuth (Dynamic)
Cursor supports built-in OAuth for servers that require it. Click "Add to Cursor" for one-click OAuth setup.
OAuth (Static Credentials)
Provide fixed CLIENT_ID and optional CLIENT_SECRET in the auth block. For providers requiring whitelisted redirect URLs.
Header-Based Auth
Set Authorization or custom headers in the headers field for remote servers. Supports interpolation.
OAuth Redirect URLCursor uses a fixed redirect URL for all MCP servers: cursor://anysphere.cursor-mcp/oauth/callback — register this in your OAuth provider's allowed redirect URIs.
Static OAuth Fields
FieldRequiredDescription
CLIENT_IDYesOAuth 2.0 Client ID from provider
CLIENT_SECRETNoClient Secret (for confidential clients)
scopesNoOAuth scopes; auto-discovered via well-known endpoint if omitted
When to Use Static OAuthUse when the provider gives you a fixed Client ID, requires whitelisting a redirect URL (e.g. Figma, Linear), or doesn't support Dynamic Client Registration.
Using MCP in Chat
Agent & Tool Usage

Agent automatically uses MCP tools listed under "Available Tools" when relevant — including in Plan Mode. You can also ask for a specific tool by name.

User Prompt
→
Agent Selects Tool
→
Approval Check
→
Tool Executes
→
Result in Chat
Toggle Tools
Click a tool name in the tools list to enable/disable it. Disabled tools won't load into context or appear to Agent.
Auto-Run
Enable auto-run for Agent to use MCP tools without asking for approval each time. Works like terminal command auto-run.
Image SupportMCP servers can return base64-encoded images (screenshots, diagrams). Cursor attaches them to chat and the model can analyze them if it supports vision.
Tool Approval Flow
MCP tool call triggered by Agent
|-- Is auto-run enabled?
|   |-- YES -> Execute immediately
|   +-- NO  -> Show approval prompt
|       |-- APPROVE -> Execute tool
|       +-- REJECT  -> Skip tool call
+-- Tool returns result
    |-- SUCCESS -> Show expandable response in chat
    +-- ERROR   -> Show error message, retry available
Isolated FailuresIf an MCP server crashes or times out, other MCP servers continue working normally. Cursor isolates server failures to prevent cascading issues.

Security Best Practices
Security Checklist
Verify SourceOnly install MCP servers from trusted developers and repositories. MCP servers execute code on your behalf.
Review PermissionsCheck what data and APIs the server will access before installing. Understand what the server does.
Limit API KeysUse restricted API keys with minimal required permissions. Never use admin/root keys for MCP integrations.
Audit CodeFor critical integrations, review the server's source code. Prefer open-source servers with visible repositories.
Environment Variable Safety
Correct: use env interpolation 
"env": {
  "API_KEY": "${env:MY_SECRET_KEY}"   // resolved at runtime
}
Wrong: hardcoded secret 
"env": {
  "API_KEY": "sk-abc123hardcoded"   // NEVER do this
}
Never Hardcode SecretsAlways use ${env:NAME} interpolation or envFile to load secrets. Hardcoded keys in mcp.json may leak to version control.
envFile for Local ServersUse "envFile": "${workspaceFolder}/.env" for stdio servers. Add .env to .gitignore.
Debugging & Troubleshooting
Debugging MCP Issues

View MCP logs to diagnose connection errors, authentication issues, or server crashes:

Steps to view MCP logs 
1. Open Output panel: Cmd+Shift+U (Mac) / Ctrl+Shift+U (Win)
2. Select "MCP Logs" from the dropdown
3. Check for connection, auth, or crash errors
Server ManagementToggle servers on/off without removing them: Settings (Cmd+Shift+J) → Features → Model Context Protocol → click toggle.
Common Issues & Fixes
MCP server not appearing in chat?
|-- Check -> Is server enabled in Settings → Features → MCP?
|-- Check -> Is mcp.json valid JSON? (no trailing commas)
+-- Check -> View MCP Logs for connection errors

MCP server crashes or times out?
|-- Error shown in chat, tool call marked as failed
|-- Other MCP servers continue working (isolated)
+-- Retry the operation or check logs for details

Need to update an npm-based MCP server?
|-- 1. Remove the server from settings
|-- 2. Run: npm cache clean --force
+-- 3. Re-add the server to get latest version
Server IsolationCursor isolates server failures — one crashing server won't affect others. If a tool call fails, you can safely retry.
Quick Decision Guide
Which Transport & Config Do I Need?
Is the MCP server for personal use only?
|-- YES -> Does it need to run locally?
|   |-- YES -> Use stdio + ~/.cursor/mcp.json (global)
|   +-- NO  -> Use SSE/HTTP + ~/.cursor/mcp.json
+-- NO  -> Is it for the whole team?
    |-- YES -> Use .cursor/mcp.json (project) + commit to repo
    +-- NO  -> Use global config per developer

Does the server require authentication?
|-- API Key     -> Use env field with ${env:NAME} interpolation
|-- OAuth       -> Use one-click install or auth block in mcp.json
+-- None        -> Just add command/url, no auth needed

Want to install a server from the official catalog?
|-- YES -> Click "Add to Cursor" on cursor.com/docs/context/mcp
+-- Custom -> Add manually to mcp.json with command or url field
Extension APIFor enterprise environments, use vscode.cursor.mcp.registerServer() to programmatically register MCP servers without modifying mcp.json files.
