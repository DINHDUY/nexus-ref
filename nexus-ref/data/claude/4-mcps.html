Built-In Tools — All 20 (SDK ToolInputSchemas-Verified)
🔧 Complete Built-In Tool Reference
Read
Read any file. Default 2,000 lines. Use offset + limit for large files. Multimodal — renders images. Batch multiple reads in one response.
Must Read before Edit. Lines >2,000 chars truncated.
Edit
Exact string replacement. old_string must be unique in file. replace_all to rename across entire file. Fails if match isn't unique.
Must Read first. Add surrounding context to make old_string unique.
MultiEdit
Multiple sequential edits to one file in a single atomic call. All succeed or none apply. Each edit operates on the result of the prior one.
Prefer over repeated Edit calls on the same file.
Write
Overwrite an entire file. Must Read first if file exists. Never create .md/README files unless explicitly asked.
Destructive. Prefer Edit/MultiEdit for targeted changes.
Glob
Fast file pattern matching. Patterns like **/*.ts or src/**/*.tsx. Results sorted by modification time.
Prefer over Agent for known file name patterns.
Grep
Content search with full regex. include filters by file type (e.g. *.ts). For counting matches, use Bash with rg.
Returns paths sorted by mtime, not match count.
LS
List directory. Requires absolute path. Supports ignore glob array. Prefer Glob/Grep when you know what to look for.
Verify parent exists before creating dirs.
Bash
Shell in persistent session. Default 2 min timeout; max 10 min. Output truncated at 30,000 chars. Use rg not grep. Chain with ; or &&, not newlines.
Always HEREDOC for git commits. Don't cat/ls — use Read/LS.
Agent (Task)
Spawn isolated subagent. Returns ONE final message — stateless. Params: description, prompt, subagent_type, model, resume, run_in_background, max_turns, name, mode.
Batch multiple Agent calls in ONE message for concurrency.
TaskStop
Stop a running background task or shell by ID. Used when run_in_background: true was set on an Agent call that needs to be cancelled.
Background tasks only. Regular Agent calls are synchronous.
NotebookRead
Read Jupyter (.ipynb). Returns all cells + outputs. Absolute path required. Use instead of Read for notebooks.
Read cannot parse .ipynb format.
NotebookEdit
Edit a cell. Params: notebook_path, cell_id (0-indexed string), new_source, cell_type?, edit_mode: replace/insert/delete.
cell_type required for insert mode.
WebFetch
Fetch URL → process with fast model. HTML → markdown. 15-min cache. Binary responses (PDFs etc.) saved to disk. Prefer mcp__* fetch tool if available.
HTTP auto-upgrades to HTTPS. Read-only.
WebSearch
Search the web. allowed_domains / blocked_domains filtering. Returns search result blocks. US only.
For post-cutoff info only. Not available outside US.
TodoRead
Read session todo list. Takes no parameters. Use proactively: session start, before tasks, when uncertain, after completions.
Call frequently — it's free and keeps you on track.
TodoWrite
Manage task list. todos[] with content, status (pending/in_progress/completed), priority (high/medium/low), id. One in_progress at a time.
Use for 3+ step tasks. Mark completed immediately when done.
ListMcpResources
List all available MCP resources from connected servers. Returns the full catalog of @server:type://path resources you can reference.
Call when you need to discover what data sources are available.
ReadMcpResource
Read a specific MCP resource by URI. Same data as @server:type://path mention but programmatic — useful inside Agent subagents.
Equivalent to typing @resource in the prompt.
SubscribeMcpResource
Subscribe to a resource for live updates via MCP list_changed notifications. Claude Code is notified when the resource changes.
Pair with UnsubscribeMcpResource to clean up.
AskUserQuestion
Ask the user a clarifying question and wait for their reply. Surfaces a prompt in the UI without the overhead of a full conversation turn.
For mid-task clarification when critical info is missing.
ℹ Additional internal tools (not typically user-facing) Config (get/set settings), ExitPlanMode (plan mode only), EnterWorktree (git worktree), SubscribePolling / UnsubscribePolling (resource polling fallback when list_changed not supported), TaskOutput (background task result retrieval) — all verified from SDK ToolInputSchemas union type.

── When TO use Agent ─────────────────────────────
✓ Open-ended search: "which file handles X?" or "find all usages of Y"
✓ Parallel workstreams — batch multiple Agent calls in ONE message
✓ Background research while continuing main task (run_in_background: true)

── When NOT to use Agent ─────────────────────────
✗ Specific file path  → use Read or Glob directly (faster)
✗ "class Foo" search  → use Glob directly
✗ 2-3 known files     → use Read directly

── Stateless contract ────────────────────────────
# Returns ONE final message. Cannot send follow-up.
# Agent result NOT visible to user — you must summarize.
# Specify EXACTLY what info to return.
# Tell agent: research-only vs. write code.
🔑 Concurrency: always batch Agent calls Launch multiple independent Tasks in a single message with multiple tool uses — they run concurrently. Sequential calls are up to 10× slower.

File operation rules and critical gotchas 
── Read ──────────────────────────────────────────
# Default: 2,000 lines from line 1
# Large files: use offset + limit params
# Lines > 2,000 chars: silently truncated

# Faster than N separate Edit calls

── Write caution ─────────────────────────────────
# Overwrites entire file — destructive
# Must Read existing file first
# Prefer Edit/MultiEdit for targeted changes
# NEVER create .md / README files unprompted

MCP — What It Is & Three Transport Types
🔌 Model Context Protocol — The USB-C for AI Tools

An open standard for connecting Claude Code to external tools, databases, and APIs. MCP servers expose: Tools (functions Claude calls), Resources (data @-mentionable in chat), Prompts (templates as slash commands). Works with any language — build once, integrate everywhere.

HTTP / Streamable HTTP
Recommended
Remote servers via HTTP endpoints. Best for hosted services. Supports OAuth 2.0 and bearer token auth. Streamable HTTP is the current spec — use when available.
SSE
Deprecated
Server-Sent Events streaming. Still supported by Claude Code, but officially deprecated. Migrate to HTTP/Streamable HTTP. Third-party servers may require --transport sse flag.
stdio
Local Process
Claude Code launches the process, communicates via stdin/stdout. Best for local tools: npx packages, Python scripts, native binaries. Requires -- separator in CLI.
⚠ MCP binary content — new behavior MCP tools returning PDFs, Office documents, or audio now save decoded bytes to disk with the correct file extension instead of dumping raw base64 into the conversation context. WebFetch also saves binary responses alongside its summary. This reduces context waste enormously.
ℹ MCP tool naming convention throughout Claude Code All MCP tools are accessed as mcp__servername__toolname. A server named github with tool create_pr → mcp__github__create_pr. Use this pattern in allow/deny permission rules, disallowedTools settings, and hook matchers (mcp__github__.*).

Tool Search — how it works 
── Problem: many MCP servers → huge context ──────
── Solution: defer tool loading until needed ──────

When auto triggers (>10% context window):
  → Tools marked defer_loading: true (not in context upfront)
  → Claude uses MCPSearch tool to find what it needs
  → Only needed tools loaded into context
  → From your view: tools work exactly as before

── Model requirement ─────────────────────────────
⚠ Requires tool_reference block support:
  Sonnet 4+ or Opus 4+ only
  Haiku models: Tool Search NOT supported

Decision Tree — Which Tool to Use
🌿 Tool Selection Decision Tree
Need to find something in the codebase?
├─ Know the exact file path?
│  └─ YES → .ipynb file? NotebookRead  /  Otherwise: Read  (batch multiple in one call)
│  └─ NO  → Know the file name pattern (e.g. *.ts, *config*)?
│         ├─ YES → Glob  (fast, sorted by mtime)
│         └─ NO  → Know a regex or keyword inside the file?
│               ├─ YES → Grep  (regex search + file type filter)
│               └─ NO  → Open-ended, may take multiple rounds?
│                     └─ YES → Agent (Task)  (batch multiple concurrently)

Need to modify a file?
├─ Single targeted replacement?   → Edit         (must Read first; old_string unique)
├─ Multiple changes, same file?   → MultiEdit    (atomic batch)
├─ Rewrite entire file content?   → Write        (must Read first; destructive)
└─ Edit a Jupyter cell?           → NotebookEdit (modes: replace/insert/delete; cell_id 0-indexed)

Need to run a command?
├─ Shell command or script?       → Bash  (use rg; chain with ; or &&; HEREDOC for commits)
└─ List a directory?              → LS    (absolute path; prefer Glob/Grep when possible)

Need external / web data?
├─ Have the URL?                   → WebFetch  (prefer mcp__* fetch tools if available)
├─ Need to search the web?         → WebSearch (US only; use domain filters)
├─ External service (GitHub, DB)?  → MCP tool  mcp__servername__toolname
└─ Browse available MCP resources? → ListMcpResources or type @ in prompt

Need to track tasks?
├─ 3+ steps or complex task?      → TodoWrite  (plan first; one in_progress at a time)
├─ Single simple task?             → skip TodoWrite — just do it
└─ Unsure what's pending?          → TodoRead   (call proactively and frequently)

Need to clarify something with the user mid-task?
└─ Yes, need specific info before proceeding → AskUserQuestion  (surfaces in UI cleanly)
