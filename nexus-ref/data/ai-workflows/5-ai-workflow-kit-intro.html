Module 2 — Install · 3 min
Three Ways to Get Started
Global install, one-off npx, or straight from GitHub — plus the full set of CLI options for the add-workflows command.
Option A — Global install (recommended for frequent use)
npm install -g @dtranllc/ai-workflow-kit # then use anywhere: ai-workflow-kit add-workflows <workflowName>
Option B — Run without installing (npx)

Option C — Straight from GitHub

⚠
Requires Node.js 22 or later. Check your version with node --version before running.

Module 4 — In Practice · 3 min
Running Real Commands
Common patterns: adding a workflow, previewing before committing, targeting a custom output path, and using --force on updates.
Command template
npx @dtranllc/ai-workflow-kit add-workflows [workflowName] [--output path] [--dry-run] [--force]
Add the builder workflow to Cursor
npx @dtranllc/ai-workflow-kit add-workflows builder --output .cursor/agents
Preview before writing (dry-run)
npx @dtranllc/ai-workflow-kit add-workflows perf --dry-run # prints which files would be written — nothing touches disk
Overwrite existing files on update
npx @dtranllc/ai-workflow-kit add-workflows nextjs --force
Use Claude Code agents instead of Cursor
npx @dtranllc/ai-workflow-kit add-workflows speckit --output .claude/agents
✓
After running: Agent files are in your output dir (e.g. .cursor/agents/), docs land in docs/workflows/<name>/. Open your agent runner and invoke by name.

Module 5 — Mastery · 3 min
Power Use & Contribution
Tips for getting the most from the kit, using dev mode for local development, and extending with your own workflows.
①
Always dry-run first on existing projects
Use --dry-run before adding to a project that already has agent files. It prevents accidental overwrites of customized agents.
②
Use builder to create domain-specific workflows
Add the builder workflow, then invoke it with a domain description. It will research, decompose, and generate a production-ready workflow with docs.
③
Target the right agent runner
Use --output .cursor/agents for Cursor, --output .claude/agents for Claude Code. The kit is runner-agnostic.
④
Use dev mode for local iteration
After cloning the repo, use make dev ARGS="..." to run the CLI from source. Enables rapid workflow authoring without publishing.
Local development workflow
git clone https://github.com/DINHDUY/ai-workflow-kit.git cd ai-workflow-kit && npm install # Run CLI in dev mode with any args: make dev ARGS="add-workflows builder --output /tmp/test --dry-run" # Other targets: make build # compile TypeScript → dist/ make test # run all tests make typecheck # type-check without emitting
