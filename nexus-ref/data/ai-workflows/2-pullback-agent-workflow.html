Module 1 — What this system does · 3 min
The Pullback Trading Agent: a 6-phase AI pipeline
This system automates the full day-trading workflow for pullback/continuation setups — from pre-market scanning at 8 AM to post-trade journaling after market close, using seven coordinated Claude subagents.
The problem it solves
Manual scanning is slow and emotional
A trader typically spends 60–90 minutes pre-market scanning 50+ tickers. The agent compresses this to seconds, using consistent, rules-based criteria every time.
The workflow covered
Pre-market to journal — end-to-end
Scan → Analyze setup → Confirm entry → Size position → Manage trade → Journal outcome. Each phase is handled by a dedicated subagent with a focused role.
The trading style
VWAP / EMA pullback on high-RVOL stocks
Targets stocks gapping up with a clear catalyst, pulling back to VWAP or 9/20 EMA support, then forming a bullish reversal candle to enter long.

Module 2 — Meet the 7 agents · 3 min
One orchestrator, six specialists
Each agent has a single responsibility. The orchestrator coordinates all six specialists in sequence. Two agents use o3-mini for pure math/logic tasks; the rest use Claude Sonnet for reasoning and language.
Agent Model What it does
pullback.orchestrator sonnet-4-5 Entry point — coordinates all 6 phases end-to-end
pullback.scanner sonnet-4-5 Pre-market screener — builds daily watchlist from gap, RVOL, and catalyst data
pullback.setup-analyzer sonnet-4-5 Grades trend quality, flagpole strength, and pullback depth against Fibonacci levels
pullback.entry-advisor sonnet-4-5 Confirms bullish reversal candle pattern and volume expansion at VWAP/EMA support
pullback.risk-manager o3-mini Calculates exact position size, ATR-adjusted stop, and 2R–4R profit targets
pullback.trade-manager sonnet-4-5 Manages open trades — breakeven at +1R, Chandelier Exit trailing, scale-outs at 2R
pullback.journal o3-mini Post-trade log — A–F quality grade, R-multiples, win rate, and session avg R:R
Model selection rationale
Claude Sonnet handles tasks needing language comprehension, market reasoning, and nuanced judgment — like interpreting chart structure or reading candle patterns.

o3-mini handles deterministic math — position sizing and R-multiple calculations don't need prose, they need accuracy. Using a reasoning-focused model reduces arithmetic errors on the most consequential outputs.

Module 4 — Calling agents in practice · 3 min
How you actually invoke the system
You can run the full pipeline via the orchestrator, or call individual subagents for focused tasks. Here are the four most common invocation patterns with real examples.
Full pipeline — orchestrator
Run agent /pullback.orchestrator pipeline for today. Account size: $120,000. Prioritize NVDA and TSLA if they gap up.
Pre-market scan only
@pullback.scanner Scan for pullback candidates for today. Show top 5 by catalyst strength and RVOL.
Setup analysis for a specific ticker
@pullback.setup-analyzer Analyze NVDA for a pullback setup. Current price: $875. VWAP: $862. 9 EMA: $858. ATR: $14.20. Pre-market high was $882.
Trade management — get chandelier exit update
@pullback.trade-manager I'm long TSLA from $248.00, stop at $244.50. Current price: $256.80. Highest high since entry: $257.40. ATR: $3.60. Update my Chandelier Exit and advise on stop placement.
Post-trade journal entry
@pullback.journal Log this trade: NVDA long, entry $870, stop $858, exit $895 (full) at 10:45 AM. ATR was $14. RVOL was 3.2×. Setup had bullish engulfing off VWAP. Account $50,000.

R1
Move stop to breakeven at +1R
Once the position is up by 1 full R, the trade-manager moves the stop to your entry price. You can never lose on this trade again.
R2
Scale out 50% at +2R
At 2R gain, the agent recommends selling half the position. This locks in profit and reduces pressure to hold the remainder.
CX
Chandelier Exit for the runner
The remaining 50% is trailed using highest-high − (ATR × 3). This lets big moves run without giving up too much on the way back down.
JR
Journal grades the whole process, not just the outcome
A trade that hit 3R but had a D-grade setup is still graded D. The journal forces you to separate execution quality from lucky outcomes.
