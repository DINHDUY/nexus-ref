Module 1 — What is an agent? · 5 min
Welcome: from chatbot to coworker
You already know how to chat with Claude. An agent workflow takes that one step further — Claude doesn't just answer, it acts.
Regular Claude chat
You ask one question
Claude answers immediately
You do the next step
One turn at a time
Agent workflow
You give a goal
Claude plans + takes steps
Claude uses tools to act
Loops until goal is done
"Think of regular Claude like a very smart consultant you can call for advice. An agent workflow is that same consultant — except now they can also open your laptop, write the email, and send it."
Quick check — which of these is an agent-style task?
A"What's the capital of France?"
B"Summarise this paragraph for me"
C"Find all overdue invoices in my inbox, draft a follow-up for each, and add them to a spreadsheet"
Module 2 — How it works · 7 min
The agent loop: think, act, observe, repeat
Every agent workflow runs a continuous loop. Understanding the loop helps you know what's happening when an agent is "thinking".
Step 1
Receive goal
You describe what you want
→
Step 2
Plan
Claude breaks it into steps
→
Step 3
Use a tool
Search, write, click, read
→
Step 4
Observe result
What did the tool return?
→
Step 5
Decide
Done? Or next step?
Live example — "Summarise my emails about Project X and add action items to Notion"
Tool call: search_email(query="Project X", limit=20)
Result: 7 emails found. Subjects: "Project X kickoff", "X timeline", "X budget Q2"…
I found 7 emails. Let me read each one and extract any action items...
Tool call: create_notion_page(title="Project X actions", content=[...])
Done. I've created a Notion page with 5 action items grouped by owner. Want me to send a summary to the team?
The key insight
The agent doesn't just generate text — it observes the real world (email, files, web) and takes real actions (write, create, send). You stay in control by deciding when to let it proceed.
Module 3 — Building blocks · 5 min
Four ingredients every agent needs
You don't need to build these — but knowing them helps you design better workflows and spot problems faster.
◈
1. The model
Claude is the brain. It reads your goal, reasons about it, and decides what actions to take next. You set personality and constraints via a system prompt.
◉
2. Tools
What Claude can reach out and touch — web search, Gmail, Google Drive, code execution, file creation, calendar. Each tool is a capability you grant.
◫
3. Memory / context
What the agent knows: past steps in this run, your preferences, documents you've shared. Context = how informed the agent is.
◳
4. Human checkpoints
Moments where the agent pauses and asks for your approval before a risky action — like sending an email or deleting a file.
Hands-on checklist — design a simple agent
Decide the goal: what task should the agent complete?
List the tools it needs (e.g. Gmail access, web search)
Write a clear system prompt: who is this agent and what are its rules?
Identify at least one checkpoint: where must Claude ask before acting?
Test with a simple case before giving it real data
Module 4 — Real-world use cases · 5 min
Where agent workflows shine
The best use cases share a pattern: repetitive, multi-step work that requires judgment — not just data retrieval.
Research assistant
"Every Monday, find the top 5 AI news stories, summarise them in plain English, and email them to my team by 8 am."
Tools needed: web search · Gmail · scheduling · Claude summarisation
HR operations
"When a new employee form is submitted, create their Slack account, add them to the right Google Groups, and draft a welcome email from their manager."
Tools needed: Google Forms trigger · Slack API · Google Admin · Gmail
Customer support triage
"When a ticket arrives, classify the urgency, look up the customer's history, draft a personalised response, and route it to the right agent."
Tools needed: CRM read · ticket system · email draft · routing logic
Finance reconciliation
"Each week, pull invoices from Google Drive, match them to payments in the spreadsheet, flag mismatches, and send a status report."
Tools needed: Google Drive · Sheets · email · file comparison
Common patterns across all these examples:
Triggered by an event Multi-step reasoning Crosses multiple tools Produces a real output Needs occasional human check
Module 5 — Best practices · 5 min
Making your agent reliable and safe
A well-designed agent is a time-saver. A poorly-designed one creates more work. These five principles separate the two.
01
Start with a narrow scope
Give the agent one job first. A focused agent is easier to test and fix than one doing five things at once.
02
Be specific in your system prompt
Don't say "handle customer emails nicely." Say "respond within 50 words, never promise refunds, always escalate billing issues."
03
Use read-only tools first
Give the agent permission to read before it can write. Test that it reads correctly before you let it take action.
04
Add a human checkpoint for irreversible actions
Sending, posting, deleting, billing — these are irreversible. Always require confirmation before the agent proceeds.
05
Log everything and review weekly
Keep a record of what the agent did. Review it once a week for the first month. Agents improve dramatically with iteration.
