The Core Mental Model — Context as Working Memory
🍲 The Cooking Analogy

Working with AI models is fundamentally about managing the context you provide them — not just writing better prompts. Think of it like cooking a soup.

🍲 Cooking a Soup
  • Ingredients = inputs
  • Recipe = plan / path
  • Track progress as you cook
  • Result: the soup
  • Different chefs → different taste
  • Same recipe → slightly different
≈
💻 Working with AI
  • Codebase + files + prompt = inputs
  • AI plan / todo list = path
  • Check items off as tasks complete
  • Result: generated code
  • Same prompt → slightly different output
  • Context quality = output quality
🔑 Key Insight Improving model output is not primarily about better prompts — it's about managing what context the model has access to. Better context → better output.
🧠 Context = The Model's Working Memory

Think of context as a long running list that the AI model uses as its working memory for the conversation. Everything the model can "see" lives in this list.

System Prompt
Tool creator instructions — injected at the top. Defines rules, style, behaviour nudges. Set by Cursor, not by you.

⚠ Context accumulates — this matters Just like a human conversation where it gets hard to remember things said 3 hours ago, AI models have a fixed working memory. The longer the conversation, the harder it is for the model to track early details.
🚧 Context Limits

Every AI model has a context window limit — a maximum total size of context it can hold. Once reached, the model will no longer accept further messages in the conversation.

What happens at the limit 
─── Before limit ────────────────────────────────
✓ Model can see and reason about all prior turns
✓ Early instructions still in working memory
✓ Conversation flows normally

─── Near the limit ──────────────────────────────
⚠ Cursor shows context usage indicator
⚠ Early parts of conversation less reliable
⚠ Model may "forget" earlier instructions

─── At the limit ────────────────────────────────
✗ No further messages accepted
✗ Must start new conversation OR compress

─── Managing the limit ──────────────────────────
Option 1: Compress/summarize the conversation
Option 2: Start a new conversation
Option 3: Use /summarize command in Cursor
ℹ Every model has a different limit Context window sizes vary significantly across models. Larger windows cost more. AI tools like Cursor show users how close they are to the limit and provide ways to compress the conversation.
✦ Practical tip Don't wait until you hit the limit. For long coding sessions, start a new conversation per distinct task rather than one endless conversation. Smaller, focused contexts produce more reliable output.

Practical Tips for Context Management
✅ What Good Context Looks Like
✦ Be specific about the task"Add a POST /api/users route with Zod validation and return 201 on success" beats "add a user route".
✦ Attach the right filesOpen the files directly relevant to your task. Cursor auto-includes open files — use this intentionally.
✦ Include error outputPaste terminal errors or linter output. The model can read and reason about exact error messages far better than descriptions of them.
✦ Use imagesScreenshot a UI mockup, diagram, or error screen. Multimodal models include image content as text in context automatically.
✦ One task per conversationFocused conversations with tight context windows produce more reliable, on-task output than sprawling multi-hour sessions.
❌ Common Context Mistakes
⚠ Vague prompts"Fix my code" gives the model almost nothing to work with. Describe what's wrong, what the expected behaviour is, and what you've already tried.
⚠ Not opening the right filesIf the model needs to understand a type definition or utility function, open that file — don't assume it's in the codebase search results.
⚠ Ignoring context near the limitWhen context is nearly full, the model's reliability drops. Don't push through — compress or start fresh.
⚠ Letting history drift too farInstructions you gave 30 messages ago may effectively be "forgotten". Re-state critical constraints for long tasks.
⚠ Expecting the model to remember across sessionsEach new conversation starts with an empty context. Prior sessions are not in the model's working memory.
